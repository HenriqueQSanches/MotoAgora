Objetivo do projeto

Sistema para gestão de motos, clientes, manutenção (controle por KM) e cobranças, com alertas automáticos via WhatsApp e integração financeira (SGLock). A solução respeita a identidade da marca Moto Agora (dinamismo, rapidez e confiança).

Escopo do que será entregue

  • Cadastro: clientes e motos (placa, modelo, odômetro, KM para troca de óleo, endereço de pernoite).
  • Controle de KM: atualização de odômetro e registro de endereço de pernoite.
  • Manutenção: registro de manutenções (tipo, descrição, custo, KM). Alertas automáticos quando o KM atingir o valor programado para troca de óleo, com envio de aviso via WhatsApp.
  • Financeiro: cobranças (valor, vencimento, status: Pendente / Pago / Atrasado). Lembretes por WhatsApp (ex.: vencimento próximo). Integração com SGLock conforme documentação disponível.
  • Dashboard: visão geral com total de motos, manutenções pendentes, cobranças atrasadas e informações de pernoite.
  • Infraestrutura: ambiente com PostgreSQL, Redis e Docker para execução do sistema.

Tecnologias utilizadas

Frontend: React, TypeScript, Vite, Axios, React Query.

Backend: Python, FastAPI, SQLAlchemy, Alembic, Pydantic.

Infraestrutura: PostgreSQL, Redis, Docker.

Integrações: API WhatsApp, SGLock (financeiro).

Fluxos principais

Cadastro: Cliente → Moto → Endereço pernoite → Controle de KM.

Manutenção: KM alcançado → alerta automático → registro da manutenção.

Financeiro: cobrança criada → lembrete (ex.: vence amanhã) → envio via WhatsApp.
